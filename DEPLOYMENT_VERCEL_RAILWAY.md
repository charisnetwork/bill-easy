# Vercel + Railway Deployment Guide

This guide explains how to deploy Bill Easy with:
- **Frontend**: Vercel (React app)
- **Backend**: Railway (Node.js/Express API)

---

## Architecture

```
┌─────────────────┐      ┌──────────────────┐
│   Vercel        │      │   Railway        │
│   (Frontend)    │ ───► │   (Backend)      │
│                 │      │                  │
│  React + Vite   │      │  Node + Express  │
│  Static Hosting │      │  API + Database  │
└─────────────────┘      └──────────────────┘
       │                          │
       ▼                          ▼
  Custom Domain            PostgreSQL
  (optional)               (Railway DB)
```

## Step 1: Deploy Backend to Railway

### 1.1 Create Railway Project
1. Go to [railway.app](https://railway.app)
2. Click "New Project"
3. Choose "Deploy from GitHub repo"
4. Select your `bill-easy` repository
5. Railway will auto-detect the `backend/` folder

### 1.2 Add PostgreSQL Database
1. In Railway dashboard, click "New"
2. Select "Database" → "Add PostgreSQL"
3. Railway will automatically connect it to your backend

### 1.3 Set Backend Environment Variables

In Railway dashboard → Your Project → Variables, add:

```bash
# Database (auto-populated by Railway if using Railway Postgres)
DATABASE_URL=${{Postgres.DATABASE_URL}}

# JWT Secret (generate a strong random string)
JWT_SECRET=your-super-secret-jwt-key-min-32-chars

# Environment
NODE_ENV=production
PORT=3000

# CORS - Add your Vercel frontend URL(s)
FRONTEND_URL=https://your-app.vercel.app

# Optional: If using custom domain for backend
BACKEND_URL=https://your-app.up.railway.app
```

### 1.4 Update Backend URL in Frontend Code

**IMPORTANT**: Update the fallback URL in `frontend/src/config/api.js`:

```javascript
// Line 14: Update with your actual Railway URL
const RAILWAY_BACKEND_URL = 'https://your-app.up.railway.app';
```

Replace `your-app` with your actual Railway subdomain.

### 1.5 Deploy Backend
Railway will auto-deploy when you push to GitHub, or click "Deploy" manually.

## Step 2: Deploy Frontend to Vercel

### 2.1 Create Vercel Project
1. Go to [vercel.com](https://vercel.com)
2. Click "Add New Project"
3. Import your `bill-easy` repository
4. Configure:
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend/`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`

### 2.2 Set Frontend Environment Variables

In Vercel dashboard → Project Settings → Environment Variables, add:

```bash
# REQUIRED: Your Railway backend URL
VITE_BACKEND_URL=https://your-app.up.railway.app
```

**Note**: No quotes, no trailing slash.

### 2.3 Deploy Frontend
Vercel will auto-deploy on every push to the main branch.

## Environment Variables Summary

### Railway (Backend)

| Variable | Value | Required |
|----------|-------|----------|
| `DATABASE_URL` | Auto-generated by Railway | Yes |
| `JWT_SECRET` | Random 32+ character string | Yes |
| `NODE_ENV` | `production` | Yes |
| `FRONTEND_URL` | `https://your-app.vercel.app` | Yes |
| `PORT` | `3000` or Railway's default | No |

### Vercel (Frontend)

| Variable | Value | Required |
|----------|-------|----------|
| `VITE_BACKEND_URL` | `https://your-app.up.railway.app` | **Yes** |

## Custom Domain (Optional)

### Frontend Custom Domain (Vercel)
1. Vercel Dashboard → Project → Settings → Domains
2. Add your domain (e.g., `app.charisbilleasy.store`)
3. Follow DNS instructions
4. Update `FRONTEND_URL` in Railway with the new domain

### Backend Custom Domain (Railway)
1. Railway Dashboard → Project → Settings → Domains
2. Add your domain (e.g., `api.charisbilleasy.store`)
3. Update `VITE_BACKEND_URL` in Vercel with the new domain

## Troubleshooting

### CORS Errors (403 Forbidden)

**Problem**: `Access-Control-Allow-Origin` error in browser console

**Solution**: 
1. Check `FRONTEND_URL` in Railway matches your Vercel URL exactly
2. The backend now auto-allows all `*.vercel.app` domains
3. Check Railway logs for exact blocked origin

### Backend URL Not Found

**Problem**: `ERR_NAME_NOT_RESOLVED` for Railway URL

**Solution**:
1. Check Railway project is running (not sleeping)
2. Verify the URL in `frontend/src/config/api.js`
3. Redeploy backend if URL changed

### API Calls Failing

**Problem**: Frontend can't connect to backend

**Solution**:
1. Check `VITE_BACKEND_URL` is set in Vercel
2. Verify backend is accessible: `curl https://your-app.up.railway.app/api`
3. Check browser DevTools → Network for actual request URL
